059 — OAuth Credential Unification via pi-ai

Restoring the original architecture: pi-ai in the gateway for OAuth token management

Before vs After

❌ Current (Broken)

1 User pastes OAuth token (sk-ant-oat01-) into Settings UI
2 Token saved to provider_api_keys table as-is
3 Worker reads token at container launch → injects as ANTHROPIC_API_KEY env var
4 litellm detects sk-ant-oat prefix → sets Bearer auth + oauth-2025-04-20 header
5 Anthropic API returns 400 Bad Request — missing claude-code-20250219 beta, x-app: cli headers, and token is stale (never refreshed)
6 Claude CLI strips ANTHROPIC_API_KEY → falls back to .credentials.json → may be stale too
Root causes: Token never refreshed. Missing headers. Three competing auth paths. No single source of truth.

✅ After (pi-ai)

1 User clicks "Login with Claude" → pi-ai OAuth flow in browser
2 Gateway stores { refresh, access, expires } in DB via pi-ai
3 Worker requests API key → gateway checks expiry → auto-refreshes via refreshAnthropicToken()
4 Fresh access token returned with x-auth-mode: oauth header
5 Worker passes token to litellm with extra_headers — all required OAuth headers included
6 For coding agents: fresh .credentials.json written to container before spawn
Single source of truth: pi-ai manages refresh. Gateway serves fresh tokens. Workers and coding agents both get valid credentials.

Add OAuth routes: POST /api/v1/oauth/:provider/login and POST /api/v1/oauth/:provider/callback.

Modify Worker

backend/app/agent/api_key_resolver.py

Detect sk-ant-oat tokens. When detected, build extra_headers dict with required OAuth headers: anthropic-beta, user-agent, x-app. Return alongside API key.

Pass extra_headers from resolver through to all litellm.acompletion() calls. Currently only passes api_key.

Add pre-flight check: verify ~/.claude/.credentials.json exists and token isn't expired before spawning. Keep ANTHROPIC_API_KEY stripping — correct for Claude CLI.

Modify Worker

Before spawning containers, fetch fresh OAuth token from gateway. Write .credentials.json with fresh access + refresh tokens instead of mounting stale host file.
