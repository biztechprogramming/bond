045a Change Summary

New Tables

TableStorePurpose
NEW componentsSpacetimeDBThe component entity — global, hierarchical (self-referencing parent_id)
NEW component_resourcesSpacetimeDBComponent ↔ Resource link, per environment (where it runs)
NEW component_scriptsSpacetimeDBComponent ↔ Script link, global (how it's deployed)
NEW component_secretsSpacetimeDBComponent ↔ Secret key link, per environment (what it needs)

Renamed Tables (dropping deployment_ prefix)

Old NameNew NameRationale
deployment_environments→environmentsEnvironments are a platform concept, not deployment-specific
deployment_environment_approvers→environment_approversFollows from environments
deployment_environment_history→environment_historyFollows from environments
deployment_resources→resourcesServers/hosts are infrastructure, not deployment-specific
deployment_promotions→promotionsCleaner, still scoped by context
deployment_approvals→approvalsCould apply to non-deployment workflows
deployment_alert_rules→alert_rulesAlerting is broader than deployment
monitoring_alerts→alertsConsistent with alert_rules, not just monitoring

Modified Tables (schema changes)

Table (new name)ChangeMigration
MOD alertsAdd component_id TEXT (nullable FK → components)Additive — existing rows keep free-form component string, new rows get both
MOD alert_rulesAdd component_id TEXT (nullable FK → components), deprecate applies_to_resourcesAdditive — old rules still work, new rules use component_id

Unchanged

EntityStoreWhy
SAME Core platform tablesSpacetimeDB / SQLiteAgents, Messages, Sessions, Tasks, Emails, etc. — not affected
SAME scripts/registry/FilesystemScripts stay on filesystem; component_scripts indexes them
SAME receipts/FilesystemReceipts stay on filesystem
SAME secrets/FilesystemEncrypted YAML stays; component_secrets indexes the keys
SAME discovery/manifests/FilesystemManifests stay; components reference them via discovered_from
SAME logs/, locks/, proposals/FilesystemNo changes needed

Database Distribution

  • SpacetimeDB (Live): High-frequency coordination.
    • Core: agents, messages, sessions, tasks
    • Deployment (renamed): environments, resources, promotions, approvals, environment_approvers, environment_history, alerts, alert_rules
    • NEW (045a): components, component_resources, component_scripts, component_secrets
  • SQLite (knowledge.db): Semantic and relational history. emails, content_chunks (vectors), entities, memories, audit_log.
  • FileSystem (Deployment): Script registry, deployment receipts, discovery manifests/proposals, deployment logs, agent lock files.
  • FileSystem (Blobs): Content-addressed blobs (attachments, screenshots).
  • JSON/Encrypted (Config): bond.json for topology/settings, credentials.enc for encrypted tokens, secrets/{env}.yaml for deployment secrets.
