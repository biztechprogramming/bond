12 — Migration Plan

What we copy, what we adapt, what we rewrite, and what we build new — sprint by sprint.

Strategy

Bond is a new project that borrows heavily from four sources. We're not forking any single repo. We're cherry-picking the best modules, adapting them to fit Bond's architecture, and writing new glue code to connect them.

  Action Legend:

  COPY    — take the code as-is (minor renames/imports)
  ADAPT   — take the code, restructure to fit Bond's module layout + patterns
  REWRITE — same functionality, written from scratch to match Bond's architecture
  NEW     — doesn't exist in any source project
  SKIP    — exists in a source but we don't need it (yet)

  git init ~/bond
  
  # Create structure (from doc 10)

Risk Register

RiskImpactMitigation
Gateway ↔ Backend latencySlow agent turns if HTTP overhead is highKeep them on localhost. Use streaming responses. Benchmark early in Sprint 1.
Agent Zero's tool parsing is fragile with small modelsTool extraction failsAgent Zero's extract_tools is battle-tested. Copy its robust JSON parser. Validate with multiple models in Sprint 2.
OpenClaw channel adapters have deep coupling to OpenClaw internalsAdapting takes longer than expectedStart with Telegram (simplest). If coupling is bad, wrap in thin bridge instead of deep refactor.
sqlite-vec + sentence-transformers first-run is slowBad first experiencePre-download embedding model in setup wizard. Show progress bar.
Docker socket mount securityContainer escape → host accessDocument the risk clearly. Offer DinD sidecar as alternative. Sandbox containers run as non-root.
