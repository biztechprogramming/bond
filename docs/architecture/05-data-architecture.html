05 — Data Architecture

SQLite-first storage with vectors, full-text search, and content-addressed blobs.

EmailPipeline: schema, migrations, sqlite-vecAgentClaw: storage topologyAgent Zero: memory persistence

Storage Topology

Everything lives in a single directory per agent. SQLite is the default — zero config, single file, fast. PostgreSQL is an optional upgrade path for scale.

  ~/.bond/
  ├── bond.json                 ← configuration
  ├── data/
  │   ├── knowledge.db          ← SQLite + sqlite-vec + FTS5
  │   ├── blobs/                ← content-addressed file store
  │   │   ├── attachments/
  │   │   └── screenshots/
  │   └── credentials.enc       ← encrypted credential store
  ├── workspace/                ← agent working directory
  │   ├── AGENTS.md
  │   ├── SOUL.md
  │   ├── skills/
  │   └── memory/
  └── logs/

Schema Domains

Memory & Content

From Agent Zero's persistent memory + EmailPipeline's embedding approach.

  • content_chunks — id, source, text, embedding (vec), summary, timestamp
  • memories — id, type (fact/solution/instruction), content, embedding, created_at
  • session_summaries — session_key, summary, key_decisions, timestamp

Entities & Relationships

From AgentClaw's entity graph design.

  • entities — id, type, name, metadata (JSON), embedding, created, updated
  • relationships — source_id, target_id, type, weight, context, timestamp
  • entity_mentions — entity_id, source_type, source_id, timestamp

Email

From EmailPipeline's comprehensive email schema.

  • email_accounts — id, provider, credentials (enc), sync_mode, poll_interval
  • emails — id, account_id, message_id, subject, from, body, embedding, date
  • email_categories — email_id, category_id, confidence, source
  • category_prototypes — category_id, prototype_embedding, source
  • email_tasks — id, email_id, task_type, title, priority, status, due_date

Tasks

From AgentClaw's task engine + EmailPipeline's task extraction.

  • tasks — id, title, status, priority, due, parent_id, source_type, source_id
  • task_dependencies — task_id, depends_on_id, type

Skills & Tools

  • installed_skills — id, name, version, path, source, installed_at

Audit & Security

From AgentClaw's capability gate design.

  • audit_log — id, timestamp, command, actor, capability_used, context, result
  • capability_policies — id, context_pattern, capabilities (JSON)

Settings

From EmailPipeline's UI-editable settings system.

  • settings — key, value (JSON), value_type, category, is_secret

Search Architecture

Hybrid search combining vector similarity with keyword matching — the best of both worlds.

  Search Query
      │
      ├──▶ Embed query → sqlite-vec cosine similarity
      │    (semantic match: "budget discussions" finds "Q4 financial review")
      │
      ├──▶ FTS5 keyword search
      │    (exact match: finds specific names, IDs, dates)
      │
      └──▶ Merge & rank
           (reciprocal rank fusion, recency weighting)
           → Final results

Embedding Pipeline

From EmailPipeline: content flows through a pipeline of processors.

  1. Content saved to database with processed_at = NULL
  2. Background worker picks up unprocessed items in batches of 32
  3. Generate embeddings (local sentence-transformers or API)
  4. Store embedding + set processed_at + embedding_model
  5. For emails: run classification (cosine similarity vs category prototypes)

Classification

From EmailPipeline: zero-shot classification using embedding similarity. No LLM required for basic classification — fast and free after initial embedding.

  • Each category has a prototype embedding (seeded from description, refined from user actions)
  • New content classified by comparing embedding to all prototypes
  • Manual user categorization updates the prototype (rolling centroid)
  • Configurable threshold (default 0.3), multi-label support optional

Migrations

SQL-based migrations using golang-migrate (from EmailPipeline). Compatible with both SQLite and PostgreSQL.

  • Files: NNNNNN_description.up.sql / NNNNNN_description.down.sql
  • Run automatically on first start, or via CLI: bond migrate up

Retention & Pruning

DataRetention
Raw session transcriptsSummarized after 24h, originals pruned per config
MemoriesNever auto-pruned
Entity graphRelationships decay in weight without reinforcement
Audit logAppend-only, never pruned
EmailsUser's retention preference
Screenshots / blobsConfigurable (default 30 days)

Encryption

  • Credentials — Fernet symmetric encryption, key from environment or device keychain
  • Standard mode — SQLite WAL, no encryption. Fast.
  • Vault mode (optional) — SQLCipher for the entire database at rest
