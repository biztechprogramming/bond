Bond Architecture

A lightweight, local-first AI assistant for everyone — synthesized from the best of four projects.

OpenClaw — gateway, channels, tools, sessions AgentClaw — MediatR pipeline, upstream separation EmailPipeline — mediator pattern, embeddings, classification Agent Zero — multi-agent, skills, memory, code execution

Design Principles

🏠 Local-first

Everything runs on your machine. SQLite, local embeddings, no cloud required. Degrade gracefully when offline.

🪶 Lightweight

Single process, single database. No Docker required for basic usage. Minimal dependencies, fast startup.

🔌 Mediator-driven

All operations flow through a pipeline of behaviors — logging, auth, validation, transactions. Features never call each other directly.

🤖 Multi-agent

Agents spawn sub-agents for complex tasks. Each keeps focused context. Hierarchical delegation with clean reporting.

🧠 Persistent memory

Vector + full-text search over everything. Entity extraction, relationship graphs. The assistant remembers and learns.

🔒 Progressive trust

Capabilities expand as the user grants access. Safe defaults, explicit opt-in for powerful actions.

🧩 Skills, not hard-coding

Capabilities come from portable SKILL.md files. Install, remove, share. The agent loads what it needs.

💬 Channel-agnostic

Talk to Bond from any surface — web, terminal, messaging apps. Same assistant, same memory, any interface.

⚡ Simple config

One API key to start. Secrets encrypted automatically, never in config files. Progressive disclosure — wizard → UI → file.
