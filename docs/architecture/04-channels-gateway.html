Gateway Architecture

The Bond Gateway is a single-process WebSocket server that acts as the control plane for everything. It's taken directly from OpenClaw's proven architecture — battle-tested across 14+ messaging channels. Critically, the gateway also owns all LLM communication — the Python backend never calls LLMs directly. This preserves OpenClaw's OAuth subscription support, the killer feature that lets users authenticate with Claude Pro/Max or ChatGPT Plus/Pro subscriptions instead of API keys.

Adding a channel: Write an adapter that emits MessageReceived notifications and handles SendMessage commands. The mediator pipeline handles everything else.

Configuring a channel: Each channel has a guided setup flow in the Settings UI — paste a token, Bond validates it, encrypts it into the secret vault, and the channel is live. No JSON editing required. See 11 — Configuration for details.

Session Routing

From OpenClaw: sessions are isolated context buckets. Routing rules determine which session handles each inbound message.

  • DM → main session (if allowlisted) or pairing flow
  • Group message → group session (isolated per group, mention-gated)
  • Webhook → designated session
  • Cron → isolated session (clean context each run)

Web UI

Bond ships a built-in web interface combining the best of both:

  • From OpenClaw: Control UI for gateway management, WebChat for direct conversation
  • From Agent Zero: Rich streaming output with process groups, code blocks, step-by-step visibility, and real-time WebSocket updates

The web UI is served directly from the gateway — no separate frontend server needed.

LLM Proxy

The gateway is the sole point of contact with LLM providers. The Python backend constructs prompts (with tool definitions, context, and system instructions) and sends them to the gateway via an internal HTTP endpoint. The gateway handles authentication, streaming, and response relay.

Why the gateway owns LLM calls:

  • OAuth subscription support — pi-ai handles OAuth token refresh for Claude Pro/Max and ChatGPT Plus/Pro subscriptions. This is the #1 priority feature — it lets anyone use Bond with their existing subscription, no API key needed.
  • AuthProfileStore — manages multiple credential profiles per provider with rotation, cooldowns, failover (OAuth primary → API key fallback), and usage tracking. Already proven in OpenClaw.
  • Zero rewrite risk — pi-agent-core already handles the Anthropic/OpenAI API protocol (messages API, streaming, tool use). No need to reimplement in Python.
  • Single auth boundary — credentials stay in the gateway process, never passed to the Python backend.

Media Pipeline

From OpenClaw: unified handling for images, audio, video across all channels. Transcription hooks for audio messages, size caps, temp file lifecycle management.

Remote Access

The gateway binds to loopback by default. For remote access:

  • Tailscale Serve/Funnel — zero-config HTTPS over tailnet or public
  • SSH tunnels — for headless Linux servers
  • Password auth — required for any public exposure
