06 — Feature Map

Prioritized features and build phases — what to build, when, and why.

Feature Clusters

Features that share architecture patterns are grouped together. Building one lowers the cost of the others.

  Cluster A: "Know Me" ─── Memory, Context, Search
       │
       ▼  provides context to
  Cluster B: "Help Me" ─── Email, Tasks, Proactive Alerts
       │
       ▼  acts through
  Cluster C: "Do For Me" ─ Code Execution, Browser, Multi-agent
       │
       ▼  learns from
  Cluster D: "Be Me" ───── Personas, Patterns, Autonomy

Build Phases

Phase 1 — Foundation (weeks 1–4)

The substrate everything else depends on

Mediator Pipeline
AgentClaw + EmailPipeline
Replace direct calls with command/query dispatch. Wire existing tools as commands. Pipeline behaviors for logging, auth, validation.
Knowledge Store
EmailPipeline + AgentClaw
SQLite + sqlite-vec + FTS5. Repository pattern. Migration system. The single storage layer for everything.
Gateway + WebChat
OpenClaw
WebSocket control plane, session management, basic web UI. The shell everything plugs into.
Agent Loop
Agent Zero + OpenClaw
Basic agent turn: receive message → generate → execute tools → respond. Single agent, no hierarchy yet.
Phase 2 — Know Me (weeks 5–8)

Memory and context — the first user-visible features

Persistent Memory
Agent Zero
Save and recall facts, solutions, instructions across sessions. Vector search for semantic retrieval.
Entity Graph
AgentClaw
Auto-extract people, projects, decisions from conversations. Typed relationships. Context enrichment.
Smart Context
AgentClaw + OpenClaw
RAG retrieval per turn. Session summarization. Replace brute-force context stuffing with relevant retrieval.
Skills Engine
Agent Zero + OpenClaw
SKILL.md loading, skill discovery, managed installation. The extensibility foundation.
Phase 3 — Help Me (weeks 9–14)

Productivity integrations that make Bond useful daily

Multi-Agent
Agent Zero
Subordinate spawning, agent profiles, hierarchical delegation. Break complex tasks into focused subtasks.
Code Execution
Agent Zero
Terminal tool, sandbox mode, agent writes its own tools. The computer as a tool.
Email Triage
EmailPipeline
Multi-account sync (IMAP/Gmail/Yahoo), semantic search, classification, task extraction, daily briefing.
Task Engine
AgentClaw + EmailPipeline
CRUD + auto-extraction from conversations and emails. Priority, dependencies, due dates.
Channel Adapters
OpenClaw
Telegram, Discord, Slack, Signal, WhatsApp. Group routing, DM pairing, media handling.
Phase 4 — Do For Me (weeks 15–18)

Autonomous action capabilities

Browser Automation
OpenClaw
CDP-based browser control, snapshots, form filling, scraping.
Cron & Scheduling
OpenClaw + Agent Zero
Recurring jobs, one-shot reminders, background agent runs.
Proactive Intelligence
AgentClaw
Cross-domain alerting: task overdue + email urgent + calendar conflict → contextual notification.
Calendar Integration
EmailPipeline
Google Calendar read + event proposals. LLM proposes, user approves, app creates.
Phase 5 — Be Me (weeks 19+)

Higher autonomy, personalization, and polish

Persona Engine
AgentClaw
Identity switching: model, tone, capabilities, memory partition per persona.
Behavior Learning
AgentClaw
Routine detection, communication style profiling, predictive actions.
Companion Apps
OpenClaw
macOS menu bar, iOS/Android nodes, voice wake, talk mode.
Dashboard
AgentClaw + EmailPipeline
Visual overview: tasks, calendar, alerts, email status, agent activity.
