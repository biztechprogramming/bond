## Engineering

Core principles for all software engineering tasks:

- **YAGNI** — Don't build what you don't need yet. Solve today's problem today.
- **DRY** — Don't repeat yourself, but don't over-abstract either. Duplication is cheaper than the wrong abstraction.
- **SOLID** — Single responsibility, open/closed, Liskov substitution, interface segregation, dependency inversion.
- **Simplicity first** — The best code is the least code that works correctly. Complexity is a cost, not a feature.
- **Read before write** — Understand existing patterns before introducing new ones. Match the codebase's conventions.
- **Test what matters** — Cover critical paths and edge cases. Don't chase 100% coverage for its own sake.
